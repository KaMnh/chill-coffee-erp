# Phase 3B.2b.ii.a — Vitest Infra + Helper Unit Tests Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up Vitest as v4's first test runner and deliver ~70–75 pure-function unit tests across `cash-math`, `format`, `validation`, and `datetime` helpers.

**Architecture:** Vitest 2.x with `vite-tsconfig-paths` plugin for `@/` alias resolution. `environment: "node"` (no DOM this phase). TZ pinned to `Asia/Ho_Chi_Minh` via `test.env.TZ` in config so `Intl.DateTimeFormat` is deterministic. Tests use **explicit imports** from `vitest` (no globals — keeps `tsconfig.json` untouched and follows modern TS practice). Co-located in `__tests__/` dirs beside the source.

**Tech Stack:** Vitest 2.1, vite-tsconfig-paths 5.x, Node 22, TypeScript 5.8 strict.

---

## File Structure

**New files:**
- `vitest.config.ts` — runner config (plugin + env + node environment)
- `src/features/cash/__tests__/cash-math.test.ts` — 5 helpers, ~21 cases
- `src/lib/__tests__/format.test.ts` — 7 helpers, ~24 cases
- `src/lib/__tests__/validation.test.ts` — 6 validators, ~21 cases
- `src/lib/__tests__/datetime.test.ts` — 3 helpers with fake timers, ~9 cases

**Modified files:**
- `package.json` — add devDeps + 3 scripts
- `package-lock.json` — auto-regenerated by `npm install`
- `.gitignore` — add `coverage/` (forward-looking)

**Deviation from spec §5.2:** plan uses **explicit `import { describe, it, expect, vi } from "vitest"`** in every test file instead of globals. Reason: keeps `tsconfig.json` untouched (spec's `"types": ["vitest/globals"]` change would override TS's default implicit `@types/*` inclusion, risking subtle type errors elsewhere). Explicit imports are also the modern Vitest recommendation. The spec's intent ("tests work without per-file ceremony") is preserved — `import { describe, it, expect, vi } from "vitest"` is one line per file.

describe("toDatetimeLocal", () => {
  it("returns empty string for null", () => {
    expect(toDatetimeLocal(null)).toBe("");
  });

  it("returns empty string for undefined", () => {
    expect(toDatetimeLocal(undefined)).toBe("");

    // Note: this helper subtracts getTimezoneOffset(), which returns minutes

    // time + 420 min = wall-clock VN, then toISOString().slice(0,16).
